## Фаза 1 — MVP · *В работе (85%)*

**Цель**: запустить минимальную рабочую систему бронирования.

### Что входит

| Область | Ключевые возможности |
|---------|---------------------|
| **Конструктор зала** | Drag-and-drop расстановка столов, стены, сохранение плана в JSON |
| **Каталог табаков** | CRUD: название, бренд, крепость (1–5), вкусы, наличие |
| **Бронирование** | Выбор даты/времени/стола, ввод контактов, статусы |
| **Админ-панель** | Дашборд броней, подтверждение/отклонение, просмотр зала |
| **Безопасность** | JWT, шифрование телефонов, rate limiting, audit log, HTTPS |

### Критерии выхода из Фазы 1

- [ ] Гость проходит полный флоу бронирования через веб
- [ ] Админ видит, подтверждает и отклоняет брони
- [ ] Все данные передаются только по HTTPS
- [ ] Персональные данные зашифрованы в БД
- [ ] Audit log ведётся для всех действий администраторов

## Фаза 2 — Кальяны и QR · *Запланировано*

**Цель**: добавить предзаказ кальяна прямо у стола.

### Что входит

| Область | Ключевые возможности |
|---------|---------------------|
| **Аутентификация гостей** | OTP по SMS или Telegram Login Widget |
| **История гостя** | Профиль, история броней и заказов |
| **Конструктор кальяна** | Выбор крепости (3 уровня), фильтрация табаков, комментарий |
| **QR-коды за столом** | URL `/table/{id}` → выбор канала (веб или Telegram) |
| **Очередь кальянщика** | Список активных заказов, смена статусов |
| **Telegram-бот** | Заказ через бота, уведомления, команды |

## Фаза 3 — Уведомления и PWA · *Запланировано*

**Цель**: информировать гостей о статусах без лишних вопросов персоналу.

### Что входит

| Область | Ключевые возможности |
|---------|---------------------|
| **Telegram-уведомления** | Бронь подтверждена/отменена, напоминание за 2 часа, заказ принят |
| **Web Push** | Те же события через браузерные push-уведомления |
| **PWA** | Service Worker, установка на экран, работа офлайн (просмотр истории) |
| **Управление настройками** | Гость включает/отключает каналы независимо |

## Фаза 4 — Аналитика и рост · *Идеи*

**Цель**: предоставить данные для роста бизнеса и масштабирования.

> Тикеты пока не декомпозированы. Список — для ориентира.

- Дашборд аналитики для владельца: конверсия бронирований, популярность столов, топ-табаков
- Программа лояльности: история посещений, персональные рекомендации табаков
- Мультизаведение: один владелец → несколько точек
- Экспорт данных (CSV) для бухгалтерии и маркетинга
- Интеграция с онлайн-кассой (ФЗ-54)
- A/B тесты UI через feature flags

## Принципы приоритизации

1. **Сначала бронирование** — core value продукта, без него остальное не нужно
2. **Безопасность с первого дня** — не откладывать шифрование и rate limiting
3. **Mobile-first** — любая новая фича сначала проектируется для мобильного
4. **Один канал за раз** — не запускать новый канал (SMS, Push, Telegram) без завершения предыдущего
