# HookahBook — Project Rules

## Workflow

- Для каждого пункта плана создавай новую ветку (feature/task-xxx или похожее) и работай в ней.
- **Документация: один тикет за раз.** Написал спеку одного тикета → создал PR → дождался ревью от владельца → только после апрува переходи к следующей спеке. Не пиши несколько спек подряд без ревью.
- После каждой итерации запускай code review через `/everything-claude-code:code-review` для ревью всех изменённых файлов.
- После каждой итерации разработки обязательно проводи тесты в превью-режиме (preview_screenshot, preview_eval, preview_click, curl к API) — проверяй и фронтенд, и бэкенд.
- После каждого выполненного, проревьюенного и протестированного пункта плана — создавай PR в репозиторий (коммит + PR через gh).
